Understanding File Paths

File path ek address hota hai jo batata hai ki koi file (jaise image, video, ya HTML page) aapke project ke andar kahan rakhi gayi hai.

Types of File Paths

  • Relative Path: File ka location current HTML file ke relation me dikhata hai.
  • Absolute Path: File ka poora URL ya directory structure dikhata hai.

Relative Path Example

  

<img src="images/pic.jpg" alt="My Image">

Absolute Path Example

<img src="https://www.example.com/images/pic.jpg" alt="My Image">

Note: Relative paths jyadatar projects me prefer kiye jaate hain kyunki yeh easy maintainable hote hain.

Boilerplate Code < / >

Boilerplate code ek standard structure hai jo har HTML document ke start me likha jata hai. Yeh browser ko batata hai ki file HTML hai aur kis encoding ka use karna hai.

HTML Boilerplate Example

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Page</title>
</head>
<body>
    <h1>Hello World</h1>
</body>
</html>

UTF-8 vs ASCII

  • ASCII: Sirf English characters ko support karta hai (0‚Äì127 values).
  • UTF-8: World ke lagbhag saare characters aur emojis ko support karta hai.

Note: Isiliye modern web pages UTF-8 encoding ka use karte hain.

DIV, Class & ID

<div> ek container tag hota hai jo content ko group karne ke liye use hota hai. Isko class aur id ke sath aur powerful banaya jaata hai.

DIV Example

<div>
    <h2>This is inside a div</h2>
    <p>A div groups content together.</p>
</div>

Class Example

<div class="box">
    <p>This div has a class name 'box'</p>
</div>

ID Example

<div id="uniqueBox">
    <p>This div has a unique ID 'uniqueBox'</p>
</div>

Note: Class ko multiple elements ke liye use karte hain, jabki ID ek element ke liye unique hota hai.

Semantic Tags: Header, Main & Footer

Semantic tags content ko meaning dete hain. Browser aur developer dono easily samajh jaate hain ki page kaunse section me kya role play kar raha hai.

Example

<header>
    <h1>My Website Header</h1>
</header>

<main>
    <p>This is the main content of the page.</p>
</main>

<footer>
    <p>Copyright ¬© 2025</p>
</footer>
